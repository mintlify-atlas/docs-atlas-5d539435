---
title: "Quickstart"
description: "Set up the database and run your first import."
---

<Steps>
  <Step title="Check prerequisites">
    Before you begin, make sure you have the following available:

    - **MySQL Server** running and accessible
    - **Java JDK** installed
    - **MySQL Connector/J** (JDBC driver) with class `com.mysql.cj.jdbc.Driver` on your classpath
    - The project repository cloned locally

    <Note>
      The project source is under `projecteBD/src/basedades/`. The SQL schema is at `Model SQL BD/Taules_eleccions.sql`.
    </Note>
  </Step>

  <Step title="Create the database">
    Run the SQL schema script against your MySQL server. This creates the `mydb` schema and all 11 tables.

    ```bash MySQL CLI
    mysql -u <user> -p < "Model SQL BD/Taules_eleccions.sql"
    ```

    Or open the file in MySQL Workbench and execute it. The script drops and recreates the `mydb` schema, so run it on a fresh or dedicated instance.

    <Warning>
      The script begins with `DROP SCHEMA IF EXISTS mydb`. Any existing `mydb` database will be deleted.
    </Warning>
  </Step>

  <Step title="Configure the database connection">
    Open `projecteBD/src/basedades/Main.java` and update the JDBC connection string on line 20:

    ```java Main.java
    con = DriverManager.getConnection("jdbc:mysql://192.168.56.101:3306/mydb", "perepi", "pastanaga");
    ```

    Replace each part with your own values:

    | Placeholder | Replace with |
    |---|---|
    | `192.168.56.101` | Your MySQL server's IP address or hostname |
    | `3306` | Your MySQL port (default: `3306`) |
    | `mydb` | The database name (default: `mydb`) |
    | `perepi` | Your MySQL username |
    | `pastanaga` | Your MySQL password |

    For example, to connect to a local server:

    ```java Main.java (local example)
    con = DriverManager.getConnection("jdbc:mysql://localhost:3306/mydb", "root", "yourpassword");
    ```
  </Step>

  <Step title="Prepare the .DAT files">
    The importer reads fixed-width `.DAT` files from `projecteBD/fitxers/`. You can supply them in two ways:

    <Tabs>
      <Tab title="Place .DAT files directly">
        Copy the `.DAT` files into `projecteBD/fitxers/`. The importer will read them from there.

        ```
        projecteBD/
          fitxers/
            02021606.DAT
            03021606.DAT
            04021606.DAT
            05021606.DAT
            06021606.DAT
            07021606.DAT
            08021606.DAT
            ...
        ```
      </Tab>
      <Tab title="Use a ZIP archive">
        Place a ZIP file containing the `.DAT` files into `projecteBD/zips/`. The importer calls `TractarFitxers.descomprimirDATsZip()` at startup, which extracts any `.DAT` entries into `projecteBD/fitxers/` automatically.

        The default ZIP filename expected by the importer is:

        ```java TractarFitxers.java
        String fileName = "./zips/02201606_MESA.zip";
        ```

        If your ZIP file has a different name, update this line in `projecteBD/src/basedades/TractarFitxers.java`.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Run Main.java">
    Compile and run `Main.java`. The importer connects to MySQL and executes all 11 import functions in sequence:

    ```java Main.java
    TractarFitxers.descomprimirDATsZip();       // Extract .DAT files from ZIP

    Importacions.importarComunitatsAutonomes(); // Autonomous communities
    Importacions.importarProvincies();          // Provinces
    Importacions.importarMunicipis();           // Municipalities
    Importacions.importarEleccions();           // Election event
    Importacions.importarEleccionsMunicipi();   // Per-municipality election data
    Importacions.importarPersones();            // Candidate personal data
    Importacions.importarCandidatures();        // Parties / coalitions
    Importacions.importarCandidats();           // Candidates
    Importacions.importarVotsProvincies();      // Votes by province
    Importacions.importarVotsCandidatures_ca(); // Votes by autonomous community
    Importacions.importarVotsMunicipis();       // Votes by municipality
    ```

    Each function prints a confirmation message to stdout when it completes. For example:

    ```
    La taula de Comunitats Autonomes s'ha importat correctament.
    La taula de Provincies s'ha importat correctament.
    La taula de Municipis s'ha importat correctament.
    ...
    ```

    <Note>
      The functions must run in this order. Later imports (such as `importarCandidats`) depend on IDs inserted by earlier ones (such as `importarCandidatures` and `importarPersones`).
    </Note>
  </Step>

  <Step title="Verify the import">
    Run these queries in MySQL to confirm the data was loaded correctly.

    <CodeGroup>

    ```sql Check geographic tables
    SELECT COUNT(*) FROM comunitats_autonomes; -- Expect 17
    SELECT COUNT(*) FROM provincies;           -- Expect 52
    SELECT COUNT(*) FROM municipis;
    ```

    ```sql Check electoral tables
    SELECT * FROM eleccions;
    SELECT COUNT(*) FROM candidatures;
    SELECT COUNT(*) FROM candidats;
    ```

    ```sql Check vote tables
    SELECT COUNT(*) FROM vots_candidatures_mun;
    SELECT COUNT(*) FROM vots_candidatures_prov;
    SELECT COUNT(*) FROM vots_candidatures_ca;
    ```

    </CodeGroup>

    <Check>
      If all tables return non-zero row counts and the `eleccions` table shows the 2016-06-26 election date, the import completed successfully.
    </Check>
  </Step>
</Steps>
